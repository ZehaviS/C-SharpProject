## סקירה כללית

המערכת מיועדת לדמות ניהול של חנות/מאפייה:

- ניהול לקוחות.
- ניהול מוצרים ומלאי.
- ניהול מבצעים לפי מוצר, תאריכים וסוג לקוח.
- יצירת הזמנה, הוספת מוצרים לסל, חישוב הנחות וסיום הזמנה.
- שמירת נתונים בקבצי XML או עבודה מול מקור נתונים רשימתי.

הפרויקט מדגים הפרדה בין שכבות:

- UI - ממשק משתמש.
- BL - לוגיקה עסקית.
- DAL Facade - חוזים, ישויות נתונים ומפעל טעינה.
- DAL Implementations - מימושים בפועל של גישה לנתונים.
- Tools - כלי עזר משותפים.

## ארכיטקטורה

הפרויקט בנוי כך שה-UI אינו עובד ישירות מול קבצי הנתונים. הזרימה הכללית היא:

```text
UI
 ↓
BL API
 ↓
BL Implementation
 ↓
DAL Facade / DAL API
 ↓
DalXml או DalList
 ↓
XML files או Lists in memory
```

### שכבת UI

הפרויקט `UI` הוא פרויקט `WinExe` עם `UseWindowsForms=true`, ומכוון ל-:

```text
net8.0-windows
```

נקודת הכניסה היא `UI/Program.cs`, והיא מפעילה את `Form1`.

המסך הראשי מאפשר פתיחת מסכים עבור:

- לקוחות.
- מוצרים.
- מבצעים.
- הזמנה.
- העברת נתונים ממימוש הרשימות לקבצי XML.

### שכבת BL

שכבת `BL` מכילה:

- `BlApi` - ממשקים עסקיים:
  - `IBl`
  - `ICustomer`
  - `IProduct`
  - `ISale`
  - `IOrder`
- `BlImplementation` - מימוש הלוגיקה העסקית.
- `BO` - מודלים עסקיים שמוצגים לשכבת ה-UI.

הלוגיקה של הזמנות כוללת:

- בדיקת מלאי.
- חישוב מבצעים תקפים.
- חישוב מחיר לכל מוצר.
- חישוב מחיר כולל.
- עדכון מלאי לאחר Checkout.

### שכבת DAL

שכבת `DalFacade` מגדירה את החוזה לגישה לנתונים:

- `IDal`
- `Icrud<T>`
- ממשקים ייעודיים ללקוחות, מוצרים ומבצעים.
- ישויות נתונים תחת `DO`.
- Factory שטוען את מימוש ה-DAL לפי הקובץ `xml/dal-config.xml`.

יש שני מימושי DAL:

- `DalXml` - עבודה מול קבצי XML.
- `DalList` - עבודה מול רשימות בזיכרון.

## קונפיגורציה

בחירת מימוש ה-DAL מתבצעת בקובץ:

```text
xml/dal-config.xml
```

דוגמה:

```xml
<?xml version="1.0" encoding="utf-8"?>
<config>
  <dal>xml</dal>
  <dal-packages>
    <list>DalList</list>
    <xml>DalXml</xml>
  </dal-packages>
</config>
```

הערך בתוך `<dal>` קובע באיזה מימוש להשתמש:

| ערך | מימוש |
| --- | --- |
| `xml` | שימוש ב-`DalXml` ובקבצי XML. |
| `list` | שימוש ב-`DalList` וברשימות בזיכרון. |

לאחר שינוי הקונפיגורציה מומלץ לבצע build מחדש.

## קבצי XML

תיקיית `xml` כוללת את קבצי הנתונים:

| קובץ | תפקיד |
| --- | --- |
| `customers.xml` | שמירת לקוחות. |
| `products.xml` | שמירת מוצרים. |
| `sales.xml` | שמירת מבצעים. |
| `dal-config.xml` | בחירת מימוש DAL. |
| `data-config.xml` | שמירת מונים ליצירת מזהים חדשים. |

הקובץ `data-config.xml` כולל מונים כמו:

```xml
<ProductNum>1008</ProductNum>
<SaleNum>41</SaleNum>
```

מונים אלה משמשים ליצירת מזהים חדשים במימוש ה-XML.

## הערות פיתוח

- רוב הפרויקטים מוגדרים להוציא build לתיקייה משותפת: `bin`.
- קבצי XML מועתקים לתיקיית הפלט כדי שהאפליקציה תוכל לקרוא אותם בזמן ריצה.
- שינוי בחירת ה-DAL מתבצע דרך `xml/dal-config.xml`.
- יש להימנע מעריכה ידנית של קבצים בתוך `bin` ו-`obj`, כי הם נוצרים מחדש בזמן build.
- קיימים קבצי `.vs`, `bin` ו-`obj` בסביבת העבודה. בדרך כלל אלו קבצים שנוצרים על ידי Visual Studio או תהליך build ואינם חלק מלוגיקת המקור.

## פתרון תקלות נפוצות

### האפליקציה לא מוצאת את `dal-config.xml`

ודאו שקיימת תיקייה `xml` בתיקיית הפלט ושבתוכה נמצא `dal-config.xml`. במידת הצורך בצעו build מחדש.

### שינוי מ-XML ל-List לא משפיע

ודאו ששיניתם את הערך בתוך:

```xml
<dal>...</dal>
```

ולא רק את רשימת החבילות. לאחר מכן בצעו build והריצו מחדש.

### מוצר לא נוסף להזמנה

בדקו:

- שהמוצר קיים.
- שכמות המלאי מספיקה.
- שהכמות המבוקשת גדולה מ-0.

### מבצע לא מופעל

בדקו:

- שהמבצע שייך למוצר הנכון.
- שהכמות בסל עומדת בכמות המינימלית למבצע.
- שתאריך ההתחלה והסיום תקפים.
- שאם המבצע מיועד ללקוחות מועדון בלבד, ההזמנה מסומנת כלקוח מועדף.
